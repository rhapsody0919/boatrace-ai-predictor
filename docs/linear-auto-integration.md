# Linear自動統合ガイド

Claude Code（Cursor）での会話中に、AIが自動的にLinearチケットを作成・更新する機能です。

## 🎯 機能概要

会話の中で以下のような場合に、AIが自動的にLinearチケットを操作します：

1. **気づきやメモの記録**: 「これはメモしておきたい」→ Linearチケットを作成
2. **AI提案の管理**: AIが提案した内容 → Linearチケットとして記録
3. **実装タスクの管理**: 実装する機能 → Linearチケットを作成・更新
4. **進捗の記録**: 実装の進捗 → Linearチケットに自動更新

## 🔧 動作モード

### モード1: 自動実行モード

AIが判断して自動的にLinearチケットを作成・更新します。

**例:**
```
ユーザー: 「予測機能の精度を上げたい」

AI: [自動的にLinearチケットを作成]
✅ Linearチケットを作成しました: BOAT-123
   タイトル: 予測機能の精度向上
```

### モード2: 確認モード（推奨）

AIが「これをLinearに追加しますか？」と確認してから実行します。

**例:**
```
ユーザー: 「ログイン機能にバグがある」

AI: この内容をLinearチケットとして記録しますか？
    - タイトル: ログイン機能のバグ修正
    - 説明: ログイン時にエラーが発生する問題

[Linearに追加しますか？ (y/n)]
```

## 📋 使用方法

### 基本的な使い方

会話の中で、以下のような表現があると、AIがLinearチケットの作成を提案します：

- 「これはメモしておきたい」
- 「この機能を実装したい」
- 「バグがある」
- 「改善したい」
- 「タスクとして管理したい」

### 実装中の自動更新

実装を開始すると、AIが自動的にLinearチケットを更新します：

```
AI: 実装を開始します。
    → Linearチケット BOAT-123 を「進行中」に更新しました

AI: 実装が完了しました。
    → Linearチケット BOAT-123 を「完了」に更新しました
```

## ⚙️ 設定

### 環境変数の確認

Linear APIキーが設定されているか確認：

```bash
echo $LINEAR_API_KEY
```

設定されていない場合は：

```bash
export LINEAR_API_KEY="lin_api_xxxxxxxxxxxxx"
```

### 動作モードの選択

デフォルトでは「確認モード」です。自動実行モードに変更する場合は、会話の最初に以下を指定してください：

```
「Linearチケットは自動的に作成・更新してください」
```

## 💡 使用例

### 例1: 気づきを記録

```
ユーザー: 「予測アルゴリズムの計算が遅いことに気づいた」

AI: この気づきをLinearチケットとして記録しますか？
    - タイトル: 予測アルゴリズムのパフォーマンス改善
    - 説明: 計算が遅い問題を改善する

[Linearに追加しますか？ (y/n)]
```

### 例2: AI提案の管理

```
AI: 予測精度を向上させるために、過去データの分析を追加することを提案します。

[この提案をLinearチケットとして記録しますか？]
    - タイトル: 過去データ分析機能の追加
    - 説明: 予測精度向上のための過去データ分析機能

[Linearに追加しますか？ (y/n)]
```

### 例3: 実装タスクの管理

```
ユーザー: 「ログイン機能を実装したい」

AI: 実装タスクをLinearチケットとして作成しますか？
    - タイトル: ログイン機能の実装
    - 説明: ユーザー認証機能の実装

[Linearに追加しますか？ (y/n)]
→ y

AI: ✅ Linearチケットを作成しました: BOAT-123
   実装を開始しますか？
```

## 🔍 チケットの検索

作成したチケットを確認：

```bash
npm run linear:list
```

特定のチケットの詳細を確認：

```bash
npm run linear:get BOAT-123
```

## 📝 注意事項

- Linear APIキーが設定されていない場合、チケットの作成・更新はスキップされます
- チケットの作成・更新は、会話の文脈から自動的に判断されます
- 確認モードでは、必ずユーザーの承認を得てから実行します
