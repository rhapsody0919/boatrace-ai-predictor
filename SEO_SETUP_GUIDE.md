# SEO設定ガイド

このドキュメントでは、BoatAIのSEO最適化に必要な画像やファイルの作成手順を説明します。

## 📸 必要な画像ファイル

### 1. OGP画像（メイン）
**ファイルパス**: `/public/ogp-image.png`
**サイズ**: 1200×630px
**ファイルサイズ**: 200KB以下推奨
**用途**: SNSシェア時のサムネイル画像

**デザインガイドライン**:
- BoatAIのロゴを中央に配置
- 「AIボートレース予想」「高精度予想」などの普遍的なメッセージ
- 「完全無料」「登録不要」「実績公開中」などのキーメッセージ
- 背景はボートレースをイメージさせるグラデーション（青系）
- フォントは視認性の高いゴシック体
- **注意**: 的中率・回収率などの数値は変動するため、画像には含めない

#### 📐 Canvaでの作成手順

1. **Canvaにアクセス**
   - https://www.canva.com/ にアクセス
   - 無料アカウントでOK

2. **デザインを作成**
   - 「デザインを作成」→「カスタムサイズ」
   - 幅: 1200px、高さ: 630px を入力
   - 「新しいデザインを作成」をクリック

3. **背景を設定**
   - 左メニューから「素材」→「グラデーション」を検索
   - 青系のグラデーション背景を選択
   - または：「背景色」から青系のグラデーションを作成
   - おすすめカラー: `#1e3c72` → `#2a5298` （濃い青→明るい青）

4. **タイトルテキストを追加**
   - 左メニューから「テキスト」→「見出しを追加」
   - テキスト: `BoatAI`
   - フォント: Noto Sans JP Bold（日本語対応）
   - サイズ: 80-100px
   - 色: 白 (#FFFFFF)
   - 位置: 上部中央

5. **サブタイトルを追加**
   - 「テキスト」→「見出しを追加」
   - テキスト: `AIボートレース予想サービス`
   - フォント: Noto Sans JP Bold
   - サイズ: 50px
   - 色: 白 (#FFFFFF)
   - 位置: 中央

6. **キーメッセージを追加（複数）**
   - 「テキスト」→「本文を追加」

   **メッセージ1**:
   - テキスト: `完全無料・登録不要`
   - フォント: Noto Sans JP Bold
   - サイズ: 40px
   - 色: 黄色 (#FFD700)
   - 位置: 中央やや下

   **メッセージ2**:
   - テキスト: `実績公開中`
   - フォント: Noto Sans JP
   - サイズ: 36px
   - 色: 白
   - 位置: 下部中央

7. **アイコン・装飾を追加（オプション）**
   - 左メニューから「素材」→「ボート」「ボートレース」で検索
   - ボートレースのイラストやボートのアイコンを追加
   - サイズを調整して配置（邪魔にならない程度）

8. **ダウンロード**
   - 右上の「共有」→「ダウンロード」
   - ファイル形式: PNG
   - 「ダウンロード」をクリック
   - ダウンロードしたファイルを `ogp-image.png` にリネーム
   - `/public/` フォルダに配置

### 2. ロゴ画像
**ファイルパス**: `/public/logo.png`
**サイズ**: 512×512px（正方形）
**ファイルサイズ**: 50KB以下推奨
**用途**: 構造化データのpublisher.logo

**デザインガイドライン**:
- 背景透過PNG
- BoatAIのブランドカラーを使用
- シンプルで視認性の高いデザイン

#### 📐 Canvaでの作成手順

1. **デザインを作成**
   - Canvaで「カスタムサイズ」
   - 幅: 512px、高さ: 512px（正方形）
   - 「新しいデザインを作成」

2. **背景を透過に設定**
   - キャンバスを選択
   - 左上の「背景色」をクリック
   - 「透過」を選択（チェッカーボード柄）
   - ※ 透過PNG機能は Canva Pro が必要です
   - 無料版の場合: 白背景で作成し、後で remove.bg などで背景除去

3. **ロゴテキストを追加**
   - 「テキスト」→「見出しを追加」
   - テキスト: `BoatAI`
   - フォント: Noto Sans JP Black または Montserrat Bold
   - サイズ: 80-100px
   - 色: 青系 (#2a5298) または黒
   - 位置: 中央

4. **アイコンを追加（オプション）**
   - 「素材」→「ボート」「AI」「ロボット」で検索
   - シンプルなアイコンを選択
   - テキストの上または横に配置
   - サイズを調整（全体のバランスを見て）

5. **図形で装飾（オプション）**
   - 「素材」→「図形」→「円」を追加
   - 背景として円を配置
   - 色: 青系グラデーション
   - テキストを前面に配置

6. **ダウンロード**
   - 「共有」→「ダウンロード」
   - ファイル形式: PNG
   - 「背景を透過」にチェック（Pro版のみ）
   - ダウンロードして `logo.png` にリネーム
   - `/public/` フォルダに配置

**無料版で背景透過する方法**:
1. Canvaで白背景のロゴを作成
2. ダウンロード（PNG形式）
3. https://www.remove.bg/ にアクセス
4. 画像をアップロードして背景を除去
5. ダウンロードして使用

## 🔧 環境変数の設定

### Vercel/Netlifyでの設定（本番環境）

**重要**: Vercelで既に `VITE_GA_MEASUREMENT_ID` を設定済みの場合、追加作業は不要です。

設定方法（未設定の場合のみ）:
1. Vercelダッシュボード > プロジェクト選択
2. Settings > Environment Variables
3. `VITE_GA_MEASUREMENT_ID` を追加
4. 値にGoogle AnalyticsのMeasurement IDを入力
5. Production, Preview, Development すべてにチェック
6. Save

### .env.local ファイル（ローカル開発環境）

**ローカル開発環境では不要**: 開発時はGAを自動的にスキップする実装になっています。

もしローカルでもGAをテストしたい場合のみ:
```bash
# Google Analytics Measurement ID
VITE_GA_MEASUREMENT_ID=G-XXXXXXXXXX
```

### Google Analytics設定手順（未設定の場合）
1. [Google Analytics](https://analytics.google.com/)にアクセス
2. 新しいプロパティを作成
3. データストリーム > ウェブを選択
4. サイトURL `https://boat-ai.jp` を入力
5. Measurement IDをコピー（G-XXXXXXXXXの形式）
6. Vercelの環境変数に設定

## 📊 Google Search Console 初期設定（詳細ガイド）

### ステップ1: プロパティの登録

1. **Google Search Consoleにアクセス**
   - https://search.google.com/search-console にアクセス
   - Googleアカウントでログイン

2. **プロパティを追加**
   - 「プロパティを追加」または「今すぐ開始」をクリック
   - 2つの選択肢が表示されます：
     - **ドメイン**: `boat-ai.jp` （推奨・全サブドメイン含む）
     - **URLプレフィックス**: `https://boat-ai.jp`（特定のURLのみ）

3. **どちらを選ぶべきか？**
   - **ドメイン方式を推奨**: `boat-ai.jp`
   - 理由: www有り/無し、http/https すべてを一括管理できる
   - 確認方法: DNSレコード（少し難しい）

   - **URLプレフィックス方式（簡単）**: `https://boat-ai.jp`
   - 理由: HTMLファイルで確認できる（初心者向け）
   - 確認方法: HTMLファイルアップロード（簡単）

---

### ステップ2: 所有権の確認（2つの方法）

#### 方法A: HTMLファイル確認（簡単・推奨）

**URLプレフィックス**で登録した場合に使えます。

1. **確認ファイルをダウンロード**
   - Search Consoleで「HTMLファイル」を選択
   - `google[ランダム文字列].html` をダウンロード
   - 例: `google1234567890abcdef.html`

2. **ファイルをアップロード**
   - ダウンロードしたファイルを `/public/` フォルダに配置
   - 例: `/public/google1234567890abcdef.html`

3. **Vercelに再デプロイ**
   ```bash
   git add public/google*.html
   git commit -m "Add Google Search Console verification file"
   git push
   ```

4. **確認を実行**
   - Vercelのデプロイが完了したら
   - Search Consoleで「確認」ボタンをクリック
   - ✅ 成功すれば完了！

5. **確認後もファイルは削除しないこと**
   - Googleが定期的に再確認するため、ファイルは残しておく

---

#### 方法B: DNSレコード確認（上級者向け）

**ドメイン方式**で登録した場合に必要です。

1. **TXTレコードをコピー**
   - Search Consoleに表示されるTXTレコードをコピー
   - 例: `google-site-verification=xxxxxxxxxxxxxxxxxxxxxxxx`

2. **DNSプロバイダーで設定**

   **Vercelでドメインを管理している場合**:
   - Vercelダッシュボード > Domains > boat-ai.jp
   - DNS Records タブ
   - Add Record:
     - Type: `TXT`
     - Name: `@` または空欄
     - Value: コピーしたTXTレコード
   - Save

   **お名前.com / ムームードメインなど外部DNSの場合**:
   - それぞれの管理画面にログイン
   - DNS設定 / ネームサーバー設定
   - TXTレコードを追加
   - 値: コピーしたTXTレコード

3. **DNS反映を待つ**
   - 反映まで最大48時間（通常は数分〜数時間）
   - 確認: `nslookup -type=TXT boat-ai.jp` で確認可能

4. **Search Consoleで確認**
   - 「確認」ボタンをクリック
   - DNS反映前だとエラーになるので、少し待ってから再試行

---

### ステップ3: サイトマップの送信

所有権確認が完了したら、サイトマップを送信します。

1. **Search Consoleのダッシュボード**
   - 左メニュー > 「サイトマップ」をクリック

2. **サイトマップURLを入力**
   - 「新しいサイトマップの追加」に以下を入力:
   ```
   sitemap.xml
   ```
   - 「送信」をクリック

3. **確認**
   - ステータスが「成功しました」になればOK
   - 「取得できませんでした」エラーの場合:
     - ブラウザで `https://boat-ai.jp/sitemap.xml` にアクセスして確認
     - ファイルが存在することを確認
     - 数時間〜1日待って再送信

---

### ステップ4: 初期設定（オプション）

#### 4-1. ターゲット地域の設定

1. 左メニュー > 設定（歯車アイコン）
2. 「地域ターゲティング」
3. 「日本」を選択
4. 保存

#### 4-2. ユーザーの追加（チーム管理）

1. 設定 > ユーザーと権限
2. ユーザーを追加
3. メールアドレスと権限レベルを設定
   - オーナー: 全権限
   - フル: ほぼ全権限
   - 制限付き: 閲覧のみ

---

### ステップ5: インデックス登録をリクエスト

初期設定が完了したら、重要なページのインデックスをリクエストします。

1. **URL検査ツールを使用**
   - 上部の検索バーに以下のURLを1つずつ入力:
     - `https://boat-ai.jp/`
     - `https://boat-ai.jp/blog`
     - `https://boat-ai.jp/about`
     - `https://boat-ai.jp/faq`
     - `https://boat-ai.jp/how-to-use`

2. **インデックス登録をリクエスト**
   - 「URLがGoogleに登録されていません」と表示される場合
   - 「インデックス登録をリクエスト」をクリック
   - 1-2分待つ
   - 「インデックス登録をリクエスト済み」と表示されればOK

3. **ブログ記事も同様に**
   - 主要なブログ記事も同じ手順でリクエスト
   - 特にfeatured記事を優先

### よくある問題と解決策

#### ❌ 「サイトマップを読み取れませんでした」

**原因**: ファイルが存在しないか、フォーマットエラー

**解決策**:
1. ブラウザで `https://boat-ai.jp/sitemap.xml` を開く
2. ファイルが表示されるか確認
3. XMLフォーマットが正しいか確認
4. 404エラーの場合、ビルド・デプロイを確認

#### ❌ 「所有権を確認できませんでした」

**原因**: 確認ファイルが見つからない、DNSが反映されていない

**解決策**:
- HTMLファイル方式: ファイルが `/public/` にあるか確認
- DNS方式: `nslookup -type=TXT boat-ai.jp` で確認
- 数時間待ってから再試行

#### ❌ 「インデックスに登録されていません」

**原因**: Googleがまだクロールしていない（正常）

**解決策**:
- 数日〜2週間待つ（通常は1週間以内）
- 「インデックス登録をリクエスト」を実行
- サイトマップが正しく送信されているか確認
