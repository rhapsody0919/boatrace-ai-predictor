# 新機能アイディア - ボートレースAI予想サイト

## 📝 このサイトが提供できる価値

### 現在提供している価値
1. **リアルタイム性の高いレースデータ**
   - 1時間間隔で最新のレースデータを自動更新
   - 天候、気温、風速などのリアルタイム情報
   - 24時間いつでもアクセス可能

2. **AI予想による意思決定サポート**
   - 選手データに基づくスコアリング
   - 客観的な指標での艇番順位付け
   - 予想根拠の明示

3. **情報の整理と可視化**
   - 複数レース場のデータを一元管理
   - 見やすい表形式での選手情報表示
   - 初心者でも理解しやすいUI

4. **アクセスの手軽さ**
   - 会員登録不要
   - 無料で全機能利用可能
   - モバイル対応

---

## 🚀 ユーザーを増やすために提供すべき新機能

### 優先度: 高 🔴

#### 1. 的中実績の可視化
**目的**: 信頼性の証明とユーザーエンゲージメント向上

**機能内容**:
- 過去の予想結果と実際のレース結果の照合
- 本命予想の的中率（日別、週別、月別）
- トップ3推奨の的中率（3連単、3連複）
- グラフでの可視化（折れ線グラフ、円グラフ）

**実装方法**:
- localStorage または GitHub Pages の静的JSONファイルで履歴保存
- レース結果は別途スクレイピングして自動集計
- DBは不要、ファイルベースで実装可能

**ユーザーへの価値**:
- サイトの信頼性が数字で証明される
- 予想精度が明確になる
- 継続的に訪問する理由ができる

---

#### 2. リアルタイム通知機能
**目的**: ユーザーの再訪問を促す

**機能内容**:
- レース開始30分前の通知
- 高スコアレース（AIスコア90以上）のアラート
- 注目選手の出走通知

**実装方法**:
- Web Push API（Push Notifications）
- ユーザーの通知許可が必要
- Service Worker を使用

**ユーザーへの価値**:
- 見逃し防止
- タイムリーな投票判断
- 使い勝手の向上

---

#### 3. 予想履歴の保存と閲覧
**目的**: ユーザーの投票記録管理

**機能内容**:
- ユーザーが選んだ予想の保存（クリック記録）
- 自分の的中率の確認
- 投票金額と払戻金の管理（オプション）

**実装方法**:
- localStorage でブラウザに保存
- エクスポート機能（CSV、JSON）
- インポート機能で端末間のデータ移行

**ユーザーへの価値**:
- 自己分析が可能
- 投票戦略の改善
- データに基づく投票判断

---

### 優先度: 中 🟡

#### 4. フィルタリング・ソート機能
**目的**: ユーザーが目的のレースを素早く見つける

**機能内容**:
- レース時刻でフィルタ（午前、午後、夕方）
- AIスコア順でソート
- レース場のお気に入り登録

**実装方法**:
- React の state 管理で実装
- localStorage でお気に入り保存

**ユーザーへの価値**:
- 時間の節約
- 興味のあるレースに集中できる

---

#### 5. 選手詳細ページ
**目的**: 深い分析を求めるユーザーへの対応

**機能内容**:
- 選手名クリックで詳細ページへ遷移
- 過去の成績グラフ
- 得意コース、得意レース場の分析
- 対戦成績（特定の選手との相性）

**実装方法**:
- React Router でルーティング追加
- スクレイピングで選手個別ページのデータ取得
- キャッシュして静的ファイル化

**ユーザーへの価値**:
- より詳細な情報に基づく判断
- マニア層の満足度向上

---

#### 6. 比較機能
**目的**: 複数レースを同時に分析

**機能内容**:
- 最大3レースを並べて比較
- 選手間の比較（同じレース内）
- レース場間の傾向比較

**実装方法**:
- モーダルまたは分割画面で表示
- state 管理で複数レースを保持

**ユーザーへの価値**:
- 効率的な情報収集
- 複数の投票判断を同時に行える

---

#### 7. オッズ情報の統合
**目的**: 投票判断の材料を増やす

**機能内容**:
- 公式オッズの表示（スクレイピング）
- AIスコアとオッズのギャップ分析
- 期待値計算（オッズ × 的中確率）

**実装方法**:
- 公式サイトからオッズをスクレイピング
- リアルタイム性は不要（1時間間隔でOK）

**ユーザーへの価値**:
- より戦略的な投票が可能
- 期待値ベースの投票判断

---

### 優先度: 低 🟢（将来的な展望）

**機能内容**:

**実装方法**:

- OGP タグの最適化

**ユーザーへの価値**:
- 友人との共有
- 的中の喜びを共有

---

#### 9. コミュニティ機能
**目的**: ユーザー同士の交流促進

**機能内容**:
- 掲示板（レースごとのコメント）
- ユーザー投票（本命はどれ？アンケート）
- ランキング（的中率ランキング）

**実装方法**:
- バックエンドが必要（Firebase、Supabase など）
- または外部サービスの埋め込み（Disqus など）

**ユーザーへの価値**:
- コミュニティの形成
- 情報交換
- 楽しさの向上

---

**機能内容**:

- 広告非表示
- 詳細な統計情報へのアクセス

**実装方法**:
- サブスクリプション（Stripe など）

**ユーザーへの価値**:
